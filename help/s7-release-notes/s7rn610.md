## Scene7 IPS（影像生產系統）4.5.0 {#section-3e37ed37b6d146419f4578bd7ffeaa74}

**新功能**

* 現在透過JDK 1.8執行。
* 支援EXIF方向標籤。
* 暫時FTP檔案（如使用ProFTP建立）的上傳目錄檔案略過。
* 影像轉譯現在支援立即發佈。 這表示您轉譯的暈映資產可立即在網站上啟動，而無需起始發佈工作。 如果您有興趣使用此功能，請聯絡技術支援，請他們啟用您的帳戶以立即發佈。

**不再支援的功能**

* IPS Classic發佈驗證中的目錄伺服器重新同步選項 — 已移除。
* 您無法再設定空白或重複的中繼資料標籤欄位值。

**錯誤修正**

* 回應API `getAssets`時，[!DNL .vtt]檔案的`<metadataArray>`為空。

* 方向缺少userdata欄位。
* `getJobLogDetails`對於更新ACO檔案的作業的響應未正確指定作業詳細資訊。
* PSD主檔案已記錄為已發佈多次，每個影像伺服器。
* 資產名稱中包含無效字元，可上傳至IPS。
* 無法產生的視訊縮圖會保留為「佇列中擱置中」狀態。
* 在大型`deleteFolder`和伺服器目錄上載之間進行資料夾還原的競爭條件。
* 上傳時提前發佈時，每個發佈伺服器會記錄兩次PDF檔案。
* 上傳具有相同圖層名稱的多個PSD檔案會導致最佳化錯誤。
* PDFL光柵化客戶PDF會導致產生的影像出現非預期的白線。
* 上傳時發佈了多次視訊縮圖檔案。
* 重複的上傳檔案偵測產生兩個上傳通知，導致完成的檔案記帳流失。
* 視訊轉碼的Sidecar XML檔案更新時已記錄為已發佈多次。
* 刪除嵌套資料夾並將另一個資料夾（不在嵌套資料夾的父資料夾中，並且與嵌套資料夾同名）移動到以前嵌套資料夾的位置時，moveFolder失敗。
* 共用和本地配置中的群集狀態配置條目未通過清除清除。
* 變更API記錄剖析器以篩選電子郵件地址格式。
* 視訊處理無法正確對應`Iptc4xmpCore:Location`。
* 清除和上傳之間存在競爭條件。
* 發佈不使用`assetFile`，因此測試發佈只適用於新資產。
* 更新規則集時擲回Null指標。

## 檢視器 {#section-833ab92c91c941d2bfdc27f233f582ad}

**Dynamic Media Image Serving Viewers 5.2.2的新功能、增強功能和錯誤修正**

檢視器升級可回溯相容 — 您不需要變更現有的網頁程式碼。 不過，建議您在我們的測試環境中測試新檢視器。 以下網站提供如何設定系統以存取中繼伺服器的指示：

[https://helpx.adobe.com/experience-manager/scene7/kb/base/upgrade-management/testing-scene7-viewers-upgrade.html](https://helpx.adobe.com/experience-manager/scene7/kb/base/upgrade-management/testing-scene7-viewers-upgrade.html)

設定電腦以存取測試伺服器後，您可以檢查網站以測試升級。 如果您使用現成可用的HTML5檢視器，最佳作法是根據我們的標準測試伺服器和s7is5-preview-staging.scene7.com進行測試。

新功能和增強功能

另請參閱[檢視器參考指南](https://experienceleague.adobe.com/docs/dynamic-media-developer-resources/library/homeviewers.html?lang=en)。

## 不再支援的功能 {#section-9295ea9313c84695b19bca0e0f01a690}

* **Flash檢視器服務終止通** *知自2017年1月31日起，Adobe Scene7(現為Dynamic Media Classic)正式終止支援Flash檢視器平台。*

* 提醒：自2014年1月31日起，Scene7正式停止支援DHTML檢視器平台。
* Scene7Creative Suite擴充功能在此版本中將遭到取代，因為Creative Cloud應用程式不支援Flash面板。 內容擷取工作流程的進一步開發將重點放在Adobe Experience Manager Dynamic Media和Creative Cloud。 若為Scene7,Scene7 Publishing System中的上傳和匯出功能會保留，並可改用。
* 由於採用率低，UGC視訊已在此版本中淘汰。 設定另一個Dynamic Media Classic帳戶可作為UGC視訊的替代解決方案。 請聯絡Adobe技術支援以設定其他帳戶。
* 由於採用率低，此版本已淘汰文字連字功能。 連字字典定義了如何分割字詞和斷字，以支援最佳的文字流程。
* 由於採用率較低，2015年均淘汰了中繼資料伺服器和相似性搜尋。 當正式淘汰目標日期宣佈時，受影響的客戶將會收到Adobe技術支援的通知。
